# СРОЧНО: Исправление проблемы с праздниками

## Проблема

1. **В базе данных все еще 18 записей настроек** вместо 1 (запись `main`)
2. **Браузер загружает старую версию кода**, которая пытается парсить отдельные записи

## Решение (2 шага - ОБЯЗАТЕЛЬНО)

### Шаг 1: Удалить ВСЕ старые записи через SQL (ОБЯЗАТЕЛЬНО!)

1. Откройте [Supabase Dashboard](https://supabase.com/dashboard)
2. Перейдите в **SQL Editor**
3. **Выполните этот SQL запрос:**

```sql
DELETE FROM settings WHERE key != 'main';
```

4. **Проверьте результат:**

```sql
SELECT key FROM settings;
```

**Должна остаться ТОЛЬКО одна запись с `key = 'main'`!**

### Шаг 2: Полностью очистить кеш браузера

1. Откройте DevTools (F12 или Cmd+Option+I)
2. **Правый клик на кнопке обновления** (кружок со стрелкой)
3. Выберите **"Очистить кеш и жесткая перезагрузка"** (или "Empty Cache and Hard Reload")

**ИЛИ через настройки:**
1. Chrome Settings → Privacy → Clear browsing data
2. Выберите "Cached images and files"
3. Нажмите "Clear data"

## Почему это важно

- Старые записи мешают загрузке настроек
- Браузер кеширует старый JavaScript, который пытается парсить каждую запись отдельно
- После удаления старых записей И очистки кеша должны появиться логи с правильными данными

## Проверка после исправления

После выполнения обоих шагов, откройте консоль браузера (F12) - должны появиться логи:

```
[api-client] Settings loaded { holidaysCount: 11, customWeekendsCount: 6 }
[Settings] Settings loaded into store { holidaysCount: 11, customWeekendsCount: 6 }
```

**Если эти логи появляются, но праздники все равно не отображаются** - это другая проблема (возможно, они относятся к другому месяцу).
