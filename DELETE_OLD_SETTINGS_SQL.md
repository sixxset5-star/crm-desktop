# Удаление старых записей настроек

Старые записи настроек все еще присутствуют в базе данных и могут вызывать проблемы.

## Шаги для удаления

1. Откройте [Supabase Dashboard](https://supabase.com/dashboard)
2. Перейдите в SQL Editor
3. Выполните следующий SQL запрос:

```sql
DELETE FROM settings WHERE key != 'main';
```

4. Проверьте результат:

```sql
SELECT key FROM settings;
```

Должна остаться только одна запись с `key = 'main'`.

## Альтернативный способ (если SQL не работает)

Если RLS политики блокируют удаление через SQL, используйте скрипт:

```bash
cd /Users/rafael/crm-desktop
./run-migration.sh delete-old-settings
```

Но как показывает проверка, скрипт не удалил старые записи (возможно, из-за RLS). Поэтому **предпочтительно использовать SQL напрямую**.
