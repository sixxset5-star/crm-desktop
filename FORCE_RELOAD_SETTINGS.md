# Принудительная перезагрузка настроек

После удаления старых записей из Supabase, настройки могут не обновиться, если store уже загрузил их один раз.

## Решение

### Вариант 1: Очистить localStorage и перезагрузить страницу

1. Откройте DevTools (F12)
2. Перейдите в **Application** → **Local Storage**
3. Найдите ваш домен (например, `vercel.app` или `localhost`)
4. Очистите все данные
5. Сделайте Hard Reload: Cmd+Shift+R (Mac) / Ctrl+Shift+R (Windows)

### Вариант 2: Использовать консоль браузера

Откройте консоль (F12) и выполните:

```javascript
// Этот код попытается принудительно перезагрузить настройки
// Но для этого нужен доступ к store

// Попробуйте найти store через React DevTools или глобальный объект
// Если у вас есть доступ к useSettingsStore:
const { useSettingsStore } = await import('/src/store/settings');
useSettingsStore.getState().loadFromDisk(true);
```

**Но проще всего - просто очистить localStorage и перезагрузить страницу (Вариант 1).**

### Вариант 3: Добавить временную кнопку в UI

Можно добавить временную кнопку "Перезагрузить настройки" в страницу Settings, которая вызывает:

```typescript
useSettingsStore.getState().loadFromDisk(true);
```

Но это требует изменений в коде.

## Проверка результата

После перезагрузки проверьте консоль браузера - должны появиться логи:
- `[api-client] Settings loaded` с информацией о праздниках
- `[Settings] Settings loaded into store` с той же информацией

Если логи показывают правильные данные (11 праздников, 6 выходных), но они все равно не отображаются, проблема может быть в компоненте отображения или в том, что данные относятся к другому месяцу.
