# Как проверить мобильную версию

## Способ 1: DevTools браузера (рекомендуется)

1. **Запустите dev-сервер:**
   ```bash
   npm run dev
   ```
   Или только веб-версию (без Electron):
   ```bash
   npx vite
   ```

2. **Откройте в браузере:**
   - Перейдите на `http://localhost:5173`
   - Если Electron запустился, откройте DevTools в Electron окне

3. **Включите режим эмуляции устройства:**
   - **Chrome/Edge**: `Cmd+Shift+M` (Mac) или `Ctrl+Shift+M` (Windows/Linux)
   - Или через меню: DevTools → Toggle device toolbar (иконка телефона/планшета)

4. **Выберите устройство:**
   - iPhone SE (375x667) - для маленьких экранов
   - iPhone 12/13/14 Pro (390x844) - стандартный размер
   - iPad (768x1024) - для планшетов
   - Или задайте кастомный размер: 375px, 480px, 768px

5. **Проверьте:**
   - ✅ Кнопка меню (hamburger) появляется в левом верхнем углу
   - ✅ Sidebar выдвигается слева при клике на меню
   - ✅ Контент занимает всю ширину экрана
   - ✅ Grid-ы становятся одноколоночными
   - ✅ Модальные окна полноэкранные
   - ✅ Таблицы прокручиваются горизонтально

## Способ 2: На реальном мобильном устройстве

1. **Узнайте IP-адрес вашего компьютера:**
   ```bash
   # macOS/Linux
   ifconfig | grep "inet " | grep -v 127.0.0.1
   
   # Windows
   ipconfig
   ```
   Например: `192.168.1.100`

2. **Запустите Vite с доступом из сети:**
   ```bash
   npx vite --host
   ```
   Или добавьте в `vite.config.ts`:
   ```ts
   export default defineConfig({
     server: {
       host: '0.0.0.0', // Доступ из сети
       port: 5173,
     },
     // ... остальная конфигурация
   })
   ```

3. **Откройте на телефоне:**
   - Убедитесь, что телефон в той же Wi-Fi сети
   - Откройте в браузере: `http://192.168.1.100:5173`
   - Замените `192.168.1.100` на ваш IP-адрес

## Способ 3: Через build и preview

1. **Соберите проект:**
   ```bash
   npm run build
   ```

2. **Запустите preview:**
   ```bash
   npm run preview
   ```

3. **Откройте в браузере и используйте DevTools** (как в способе 1)

## Что проверить

### Основные элементы:
- [ ] Кнопка меню (hamburger) видна на мобильных
- [ ] Sidebar выдвигается и закрывается
- [ ] Overlay затемняет фон при открытом меню
- [ ] Меню закрывается при клике на ссылку
- [ ] Меню закрывается при клике вне его

### Адаптация контента:
- [ ] Контент занимает всю ширину (нет боковых отступов)
- [ ] Grid-ы с несколькими колонками → одна колонка
- [ ] Таблицы прокручиваются горизонтально
- [ ] Модальные окна полноэкранные
- [ ] Формы адаптированы (поля на всю ширину)

### Touch-интерфейс:
- [ ] Кнопки достаточно большие для нажатия (минимум 44px)
- [ ] Нет hover-эффектов (они не нужны на touch-устройствах)
- [ ] Скролл работает плавно

### Размеры экранов для тестирования:
- **375px** - iPhone SE, маленькие Android
- **480px** - средние Android телефоны
- **768px** - планшеты (iPad в портретной ориентации)
- **1024px** - планшеты (iPad в альбомной ориентации)

## Быстрая проверка breakpoints

В DevTools можно быстро переключаться между размерами:
- `375px` - мобильный (sidebar скрыт, меню hamburger)
- `768px` - планшет (sidebar может быть виден)
- `1024px+` - десктоп (sidebar всегда виден)

## Отладка на мобильном

Если что-то не работает на реальном устройстве:
1. Откройте DevTools на компьютере
2. Подключите телефон через USB
3. В Chrome: `chrome://inspect` → Remote devices
4. Или используйте Safari Web Inspector (для iPhone)
